算法复杂度用来衡量算法的运行效率。这里学习了下大O表示法，也叫渐近复杂度。

概念

这种方法用代码执行过程中，运行语句的最高频度的阶，来衡量算法的效果。对于给定的输入规模n，求得一个关于n的函数，然后将最高阶(忽略低阶，常数和最高阶的系数)，记作复杂度。举个例子：
输入规模：n
示例算法：计算前n项自然数的和。f(n) = n + (n-1) + … + 1
最高频度：运行加法n-1次，记作T(n) = n-1
复杂度表示：O(n),表示随着输入规模增大，算法运行时间线性增长。
在这个表示法里，还有标记Ω和Θ，分别表示算法运行的下界和准确界，我们一般用大O表示上界。即n大于某一常数m之后，必然有：
cf(n) >= T(n).
理清了概念，再看如何计算。
这里分为两种

计算规则

  • 加法规则
    T(n,m) = T1(n) + T2(n) = O (max ( f(n), g(m) )
  • 乘法规则
    T(n,m) = T1(n) T2(m) = O (f(n) g(m))
    T(n) = T1(n) T2(n) = O ( cf(n) ) = O( f(n) )

循环复杂度

循环复杂度计算比较简单，基本就是求和运算
举例：

for(i=1;i<=n;i++)      
   for(j=1;j<=n;j++) 
   {
        sum++； //最内层语句
   }

最高频度为 sum++ 的执行
T(n) = n²
O(n) = n²

递归复杂度

递归复杂度求法比较复杂。由于自身调用了自身，一般计算的时候，导出一个T(n)的递推表达式，然后用一些方法来求递推表达式。
一般方法有：带入，递推，母函数，主定理，差分方程等。
母函数和差分方程暂时没使用。
一般的使用带入，递推和主定理就可以搞定了。

  • 带入
    带入就是猜测，递推公式的两边比较简单，猜测带入求证。
    举例：T(n) = 2T(n/2) + O(n)
    猜测为O(n²)，则令T(n) = cn² ，带入方程，舍去低阶，方程左右平衡，
    则猜测为一个解。
    数学归纳法证明。
  • 递推
    递推法是重点，将等式一端展开，然后计算求和。
    举例为 Scramble String的计算。
    递推式为：
    T(n) = 2(T(1) + T(2) + ..T(n/2) + T(n/2+1) + T(n-1))
    = 2 3( T(1) + T(2) + ..T(n/2) + T(n/2+1) + T(n-2))
    =2     33( T(1) + T(2) + ..T(n/2) + T(n/2+1) + T(n-3))
    =2    33…3 (T(1))
    = 2    3^(n-1)
    求得 该题目 的递归解法复杂度为 O(3^(n-1))。

  • 主定理
    主定理可以称之为公式法，用来求形如T(n) = aT(n/b) + f(n)的复杂度。
    根据f(n)的不同有三类情况：

    1. 若对于某常数ε>0，有f(n) = O(nlogb a-ε )，则T(n) = O(nlogb a )
    2. 若f(n) = O(nlogb a )，则T(n) = O(nlogb a *logn)
    3. 若f(n) = O(nlogb a+ε)，且对于某常数c>1和所有充分大的正整数n，有af(n/b)≤cf(n)，则T(n)=O(f(n))。

常见应用：

1. 二分搜索 T(n) = T(n/2) + O(1) ==> O(logn)
2. 快排 T(n) = 2T(n/2) + O(n) ==> O(nlogn)
3. 快排最差：T(n) = T(n-1) + T(n) + O(n) ==> O(n^2)
