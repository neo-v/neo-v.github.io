select b.Name as Department,a.Name as Employee,a.Salary
from Employee a join Department b on a.DepartmentId = b.Id
where (
    select count(distinct(c.Salary))
    from Employee c
    where c.DepartmentId = a.DepartmentId and c.Salary > a.Salary
) < 3
order by a.Salary

Sql执行顺序

理解子查询，首先要知道Sql的执行顺序。每个步骤都会产生虚表，即VT。

  1. FROM 子句, 组装来自不同数据源的数据，可以认为是提供数据来源
  2. WHERE 子句, 基于指定的条件对记录进行筛选，基于全表的筛选
  3. GROUP BY 子句, 将数据划分为多个分组，进行局部筛选准备
  4. 使用聚合函数进行计算，分组之后选择。
  5. 使用 HAVING 子句筛选分组，使用聚合函数筛选。
  6. 计算所有的表达式，select结果。
  7. 使用 ORDER BY 对结果集进行排序

独立子查询

子查询与外部查询无关，一般提供outer 查询的条件数据

select ename,deptno,sal   
from emp   
where deptno=(select deptno from dept where loc='NEW YORK')；

一般还有自查询返回多行的，原理相同。

相关子查询

相关子查询，则是外部查询和内部查询相关。外部查询的每行，为内部查询提供支持。所以说，一般执行都是外部每一个记录，都运行子查询。
