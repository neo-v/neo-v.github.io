表达式的副作用

By Neo

发表于 Jun 28 2015

文章目录

表达式，expression，在各种编程语言中这个术语很常见，可以理解为一段计算，或者说对变量求值。如x*2+3。
编程语言中的表达式可以说分为两种，有副作用(side-effect)和没有副作用的。
所谓副作用，就是表达式在求值的过程过中，改变了参与的变量的值。最常见的比如赋值表达式：x = 2 x的值改变为2。
如何判断表达式是否有副作用：

  1. 通过辨认表达式中包含的相关类似赋值的操作符。比如 =，+=，++，—等
  2. 含有副作用的表达式，求值顺序不能改变。比如：X = 2的时候，++X ，X+4，分别求得为3，7。改变顺序，X+4，++X，分别求得6和3。

表达式副作用的应用：
在很多高级语言，比如c++和java，都支持有副作用的表达式。具体的应用表现在：

  1. 逗号表达式，逗号表达式的返回最后一个表达式的求值。所以它前边的表达式要有副作用，整个表达式才有意义。
  2. 语句，c++中表达式加上‘；’构成了语句。我们一般都是每次书写一条语句。比如 x = x + 2； 作用就是 将x的值改变。而表达式本身的求值舍弃。所以没有副作用的语句是没有意义的。

c和c++中有序列点的概念。两个序列点之间的求值顺序是任意的。一个语句‘；’就是一个序列点。所以一条表达式语句，最好不要有多个有副作用的子表达式，不然会产生难以捉摸的代码。比如：
b = (a++) + (a++) + (++a),可读性太差。由于求值顺序是任意的，先计算++a和后计算++a，表达式的值是不一样的。
